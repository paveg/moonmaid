# Phase 1.5: Animation System — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add step-by-step animation support for `@animate` diagrams, starting with BST insert visualization. Animated diagrams output self-contained HTML with playback controls.

**Architecture:** Extend the existing parser to handle `@animate` directives. A new `animation` package generates `AnimationTimeline` from tree operations. The renderer produces self-contained HTML (SVG + timeline JSON + inline playback JS) for animated diagrams, while static diagrams remain unchanged. TypeScript playback controls are embedded as inline script.

**Tech Stack:** MoonBit (animation logic), TypeScript (playback controls inlined as JS string), existing renderer/parser packages

**Spec:** `docs/superpowers/specs/2026-04-05-moonmaid-design.md` (Animation System section)

**Prerequisite:** Phase 1 complete (59 MoonBit tests, 9 TS tests passing)

**Security note:** The inline JS playback controller uses `textContent` for step descriptions to prevent XSS — never `innerHTML`. All user-provided strings pass through `escape_xml` (SVG) or `json_escape` (JSON) before embedding.

```moonbit
pub(all) enum AnimationAction {
  Highlight(node_id~ : String, color~ : String)
  Unhighlight(node_id~ : String)
  TraverseEdge(from~ : String, to~ : String, color~ : String)
  AddNode(node_id~ : String, x~ : Double, y~ : Double, width~ : Double, height~ : Double, label~ : String)
  RemoveNode(node_id~ : String)
  SwapNodes(a~ : String, b~ : String)
  UpdateLabel(node_id~ : String, text~ : String)
  CompareNodes(a~ : String, b~ : String)
  Annotate(text~ : String, x~ : Double, y~ : Double)
} derive(Show, Eq)

pub(all) struct AnimationStep {
  actions : Array[AnimationAction]
  description : String
  duration_ms : Int
} derive(Show, Eq)

pub(all) struct AnimationTimeline {
  steps : Array[AnimationStep]
} derive(Show, Eq)
```

**Files:**

- [ ] **Step 1: Write 7 failing tests** (see plan Task 4 above for full test code)

Tests: single insert, two inserts with comparison steps, three inserts, empty operations, duplicate skipping, deep insertion path length, AddNode coordinates validity.

- [ ] **Step 2: Implement `generate_bst_animation`**

The generator walks the BST insertion path for each value, producing:
- Root insert: 1 AddNode step
- Non-root insert: Highlight+Compare step, TraverseEdge step per node visited, then AddNode step
- Positions computed by running Reingold-Tilford layout on the final tree upfront
